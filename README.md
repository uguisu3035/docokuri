# どこでもクリニック LP（ランディングページ）

## プロジェクト概要

**健康管理業務のDX化サービス「どこでもクリニック」**の資料請求率を最大化するBtoB LPです。

### ターゲット
- 企業の人事・総務・健康経営担当者
- 企業規模：50〜1,000名（特に300〜1,000名が中心）
- 健診・ストレスチェック・再検査対応などの運用負荷に困っている担当者
- 健康経営優良法人の取得・更新を視野に入れている企業

### ゴール
**資料請求率の最大化**

### 1. FV（ファーストビュー）

**変更前:**
```
100％使われる福利厚生、あります。
福利厚生は「用意すること」が目的ではありません…
```

**変更後:**
```
【健康管理業務DX】
健康診断・ストレスチェック すべて自動化。
健診予約・結果管理・再検査対応・労基署報告…
人事・総務の業務負荷を、DX+BPOで劇的に削減します。
```

### 2. 問題提起

**変更前:** 福利厚生が使われない課題  
**変更後:** 健康管理業務の5つの課題を明示
- 健診予約の調整
- 紙・Excelでの管理
- 再検査フォローの属人化
- ストレスチェックの負担
- 労基署報告の手間

### 3. セクション順序

**変更前:**
1. 福利厚生が使われない課題
2. 利用シーン（3つ）
3. 差別化ポイント
4. 担当者の課題（業務負荷）
5. 解決策（DX+BPO）

**変更後:**
1. 健康管理業務の課題 ← **最優先**
2. 解決策（DX+BPO）← **早い段階**
3. DX機能の詳細（4つの柱）← **新規追加**
4. データ可視化の経営価値
5. 福利厚生サービス（補足）

### 4. CTAボタン文言

**変更前:**
- 「3分で分かるサービス資料を見る」
- 「資料で詳しく見る」

**変更後:**
- 「業務効率化の詳細を見る」
- 「削減効果をシミュレーションする」
- 「DX導入事例を見る」
- 「業務削減効果を確認」

## 💡 コンバージョン戦略

### ターゲット心理に合わせた設計

1. **課題認識（FV直後）**
   - 「まさにうちの課題だ」と共感させる

2. **解決策の提示（早い段階）**
   - Before/Afterで即座に効果を可視化

3. **具体性の担保（DX機能）**
   - 4つの柱で「何ができるか」を明確化

4. **定量効果の提示**
   - 「年間300時間削減」など数字で訴求

5. **経営価値の提示**
   - 決裁者を説得できる材料を提供

6. **心理的ハードル軽減**
   - 「営業電話なし」を全CTA箇所で明記

## 🚀 次のステップ（推奨）

### 優先度：高
1. **実際のシステム画面の追加**
   - ダッシュボードのスクリーンショット
   - 健診データ管理画面
   - 自動抽出機能のデモ画像

2. **導入実績・削減効果の数字**
   - 「導入企業○○社」
   - 「平均○○時間削減」
   - 「満足度○○%」

3. **お客様の声・導入事例**
   - 人事担当者のコメント
   - Before/After の具体的数字
   - 企業ロゴ

4. **業務削減シミュレーター**
   - 従業員数を入力すると削減時間を試算
   - インタラクティブなコンテンツ

### 優先度：中
5. **動画コンテンツ**
   - システムデモ動画（2分程度）
   - 担当者インタビュー

6. **比較表**
   - 競合サービスとの機能比較
   - 他の方法（手作業・他システム）との比較
